## 📋 РЕЗЮМЕ ДЛЯ НОВОЙ ВЕТКИ

**Что строим:** Гибридную систему управления недвижимостью для риэлторов.

**Ключевое решение:** Отказ от Telegram-админки в пользу PWA + Google Sheets.

**Модель распространения:** Хаб (один GitHub аккаунт Эрнеста) + индивидуальные таблицы агентов.

**Стек:**
- **База данных:** Google Sheets (вместо Supabase)
- **Фото:** GitHub (вместо Cloudinary)
- **Админка:** PWA (Progressive Web App) для мобильных агентов
- **Клиентское приложение:** Telegram Mini App (уже работает)
- **Бэкенд-логика:** Google Apps Script (прокси между приложением и таблицей)

### Модель "Хаб" (основной сценарий — 90% агентов)

```
┌─────────────────────────────────────────────────────────┐
│  GitHub аккаунт Эрнеста (777ernest888-oss)              │
│  ├── demo-miniapp-realty/ (клиентское приложение)       │
│  └── realty-admin-pwa/ (PWA-админка)                    │
└─────────────────────────────────────────────────────────┘
         │
         ▼
─────────────────────────────────────────────────────────┐
│  Агенты подключаются через agent_id                     │
│  ├── Агент №1 → agent_id=001 → своя Google Таблица      │
│  ├── Агент №2 → agent_id=002 → своя Google Таблица      │
│  └── Агент №3 → agent_id=003 → своя Google Таблица      │
└─────────────────────────────────────────────────────────┘
```

**Как это работает:**
1. Агент регистрируется через Google OAuth (1 клик)
2. Получает доступ к PWA-админке по ссылке: `https://777ernest888-oss.github.io/realty-admin-pwa/?agent=001`
3. При первом входе создаёт Google Таблицу по шаблону
4. Копирует ID таблицы в настройки PWA
5. Всё! Данные агента изолированы в его таблице.

**Премиум-сценарий (10% агентов — крупные агентства):**
- Отдельный GitHub аккаунт (white-label)
- Полная изоляция кода и данных
- Доплата за кастомизацию

##  ТЕХНИЧЕСКИЕ ДЕТАЛИ

### 1. Google Apps Script (файл `Code.gs`)

**Функция `doGet(e)` — для клиентского приложения:**
```javascript
function doGet(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var data = sheet.getDataRange().getValues();
  var headers = data[0];
  var rows = data.slice(1);
 
  // Получаем agent_id из URL
  var agentId = e.parameter.agent_id || '';
 
  // Фильтруем по owner_id и active
  var ownerIdIndex = headers.indexOf('owner_id');
  var activeIndex = headers.indexOf('active');
 
  var filteredRows = rows.filter(function(row) {
    var matchesAgent = !agentId || (ownerIdIndex !== -1 && row[ownerIdIndex] === agentId);
    var isActive = activeIndex !== -1 ? row[activeIndex] === true || row[activeIndex] === 'TRUE' : true;
    return matchesAgent && isActive;
  });
 
  // Скрываем секретные колонки
  var secretColumns = ['owner_contacts', 'commission', 'notes'];
  var secretIndexes = secretColumns.map(function(colName) {
    return headers.indexOf(colName);
  }).filter(function(index) { return index !== -1; });
 
  var cleanData = filteredRows.map(function(row) {
    return row.map(function(cell, index) {
      if (secretIndexes.indexOf(index) !== -1) return '';
      return cell;
    });
  });
 
  return ContentService.createTextOutput(JSON.stringify([headers].concat(cleanData)))
    .setMimeType(ContentService.MimeType.JSON);
}
```

**Функция `doPost(e)` — для PWA-админки:**
```javascript
function doPost(e) {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var headers = sheet.getRange(1, 1, 1, sheet.getLastColumn()).getValues()[0];
  var postData = JSON.parse(e.postData.contents);
  var action = postData.action; // 'create', 'update', 'delete'
 
  if (action === 'create') {
    var newRow = headers.map(function(header) {
      return postData.data[header] || '';
    });
    sheet.appendRow(newRow);
    return ContentService.createTextOutput(JSON.stringify({success: true}))
      .setMimeType(ContentService.MimeType.JSON);
  }
 
  // ... update и delete аналогично
}
```

## 📚 ИСТОРИЯ ПРИНЯТИЯ РЕШЕНИЙ

### Почему отказались от Supabase:
- ❌ Telegram WebView на мобильном не справлялся с Supabase SDK (зависания, таймауты)
- ❌ Сложность настройки для агентов (нужно создавать аккаунт, понимать SQL)
- ✅ Google Sheets проще, знакомее, бесплатнее

### Почему отказались от Cloudinary:
- ❌ Дополнительная настройка (аккаунт, API-ключи, подписи)
- ✅ GitHub уже работает, 1 ГБ хватит на 2000 фото
- ✅ Длинные URL не проблема (их никто не видит)

### Почему PWA вместо Telegram-админки:
- ❌ Telegram WebView нестабилен для тяжелых операций (загрузка фото, формы)
- ✅ PWA работает в обычном браузере (Chrome/Safari) — стабильно
- ✅ PWA устанавливается на экран как приложение

### Почему модель "Хаб":
- ✅ Один код — одно обновление (не нужно обновлять 100 копий)
- ✅ Агенты не заморачиваются с GitHub
- ✅ Вы контролируете версию
